# Canon Rules

Rules governing what is and is not considered canon in the tSAC universe.

---

## What Is Canon

Canon is the set of facts about the tSAC universe that are treated as true and authoritative. Canon includes:

- Content explicitly confirmed by the human collaborator
- Content in files within `CANON/` that has not been superseded
- Rules and mechanics in `SYSTEM/` that have been approved

---

## What Is Not Canon

The following are **not** canon unless explicitly promoted:

- Content in `SCRATCHPAD/` — this is a working area only
- Inferences drawn by an AI model during a session, even if plausible
- Story outlines in `STORY/` until marked as approved
- Anything described as "a possibility," "an option," or "a suggestion"

---

## Canon Promotion Process

To promote content from scratchpad or suggestion to canon:

1. The human collaborator explicitly states: *"This is now canon"* or moves the content into a `CANON/` file
2. The content is written into the appropriate `CANON/` file with a note of when it was confirmed
3. Any conflicting older canon is updated or superseded with a note

---

## Contradiction Handling

When a contradiction is detected:

- **Do not** silently resolve it in favor of one version
- **Do** flag it explicitly: *"Contradiction detected between [File A] and [File B]"*
- Present both versions and ask the human collaborator to adjudicate
- Once resolved, update the relevant canon file and note the resolution

---

## AI Model Behavior

AI models working in this project must:

- Treat canon files as ground truth
- Label all inferences and new suggestions clearly
- Never assert new lore as fact without explicit confirmation
- Escalate uncertainty rather than guess
- Respect the sensitivity level of this project (private)

---

## Canon Versioning

When a canon file is materially updated:

- Note the change at the bottom of the file with a brief description and date
- If the change contradicts or supersedes previous content, explicitly call it out
